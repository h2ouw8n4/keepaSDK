Keepa API

The Keepa API offers numerous endpoints. Every request requires your API access key as a parameter. You can find and change your key in the keepa portal. All requests must be issued as a HTTPS GET and accept gzip encoding. If possible, use a Keep-Alive connection. Multiple requests can be made in parallel to increase throughput.
More information: https://keepa.com/#!api

Category

Up 
get /category
Returns Amazon category information from Keepa API. (category)
Retrieve category objects using their node ids and (optional) their parent tree.

Query parameters

key (required)
Query Parameter — Your private API key. default: null
domain (required)
Query Parameter — Integer value for the Amazon locale you want to access. (1-com, 2-co.uk, 3-de, 4-fr, 5-co.jp, 6-ca, 7-cn, 8-it, 9-es, 10-in, 11-com.mx, 12-com.br, 13-com.au) default: null format: int32
category (required)
Query Parameter — The category node id of the category you want to request. For batch requests a comma separated list of ids (up to 10, the token cost stays the same). Alternatively you can specify the value 0, which will retrieve a list of all root categories. default: null format: int32
parents (required)
Query Parameter — Whether or not to include the category tree for each category. (1 = include, 0 = do not include.) default: null format: int32

Return type

array[category]

Example data

Content-Type: application/json
{
  "catId" : 281052,
  "parent" : 502394,
  "children" : [ 3109924011, 7242008011, 3017941, 2476680011, 330405011, 2476681011, 3109925011 ],
  "highestRank" : 6549,
  "name" : "Digital Cameras",
  "productCount" : 1522,
  "domainId" : 1
}

Up 
get /product
Retrieve the product for the specified ASIN and domain. (product)
Retrieves the product object for the specified ASIN and domain. If our last update is older than one hour it will be automatically refreshed before delivered to you to ensure you get near to real-time pricing data. You can request products via either their ASIN (preferred) or via UPC and EAN codes. You can not use both parameters, asin and code, in the same request. Keepa can not track Amazon Fresh and eBooks.

Query parameters

key (required)
Query Parameter — Your private API key. default: null
domain (required)
Query Parameter — Integer value for the Amazon locale you want to access. (1-com, 2-co.uk, 3-de, 4-fr, 5-co.jp, 6-ca, 7-cn, 8-it, 9-es, 10-in, 11-com.mx, 12-com.br, 13-com.au) default: null format: int32
asin (optional)
Query Parameter — The ASIN of the product you want to request. For batch requests a comma separated list of ASINs (up to one hundred). default: null
code (optional)
Query Parameter — The product code of the product you want to request. We currently allow UPC, EAN and ISBN-13 codes. For batch requests a comma separated list of codes (up to one hundred). Multiple ASINs can have the same product code, so requesting a product code can return multiple products. default: null

Return type

array[category]

Example data

Content-Type: application/json
{
  "catId" : 281052,
  "parent" : 502394,
  "children" : [ 3109924011, 7242008011, 3017941, 2476680011, 330405011, 2476681011, 3109925011 ],
  "highestRank" : 6549,
  "name" : "Digital Cameras",
  "productCount" : 1522,
  "domainId" : 1
}

Integer The category node id used by Amazon. Represents the identifier of the category. Also part of the Product object’s categories and rootCategory fields. Always a positive Long value. Note - The id 9223372036854775807 (max signed long value) denotes a blank category with the name “?”. We use this in cases where a product is listed in no or non-existent categories. format: int32
